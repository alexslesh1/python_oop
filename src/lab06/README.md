## 2. Описание реализованных типов и контейнеров

### Generic-класс TypedCollection

Класс `TypedCollection[T]` — обобщённая версия коллекции из ЛР-2. Позволяет создавать типизированные коллекции, которые хранят объекты только одного типа.

| Метод | Аннотация | Описание |
|-------|-----------|----------|
| `add(item: T) -> None` | - | Добавление элемента |
| `remove(item: T) -> None` | - | Удаление элемента |
| `remove_at(index: int) -> T` | - | Удаление по индексу |
| `get_all() -> List[T]` | - | Получение всех элементов |
| `__len__() -> int` | - | Количество элементов |
| `__getitem__(index: int) -> T` | - | Доступ по индексу |
| `__iter__()` | - | Итерация в цикле for |
| `find(predicate: Callable[[T], bool]) -> Optional[T]` | оценка 4 | Поиск первого подходящего |
| `filter(predicate: Callable[[T], bool]) -> List[T]` | оценка 4 | Фильтрация |
| `map(transform: Callable[[T], R]) -> List[R]` | оценка 4 | Преобразование с изменением типа |

### TypeVar

| TypeVar | Назначение |
|---------|------------|
| `T` | Основной тип для элементов коллекции |
| `R` | Тип результата при преобразовании (может отличаться от T) |
| `D` | Тип с ограничением `bound=Displayable` (только объекты с методом `display`) |
| `S` | Тип с ограничением `bound=Scorable` (только объекты с методом `score`) |

### Protocols (оценка 5)

| Protocol | Метод | Назначение |
|----------|-------|------------|
| `Displayable` | `display() -> str` | Объекты, которые можно отобразить |
| `Scorable` | `score() -> float` | Объекты, которые имеют оценку/балл |

### Структурная типизация

Классы не наследуются от Protocols явно. Достаточно, что у них есть нужные методы:

```python
class DisplayableBusAdapter:
    def display(self) -> str:   # Есть метод display
        return f"{self._bus.bus_id}: ..."
```

## 4. Вывод

В ходе выполнения лабораторной работы были созданы:

**Generic-класс TypedCollection[T]** — типизированная коллекция, которая хранит объекты только одного типа. Это позволяет избежать ошибок, связанных с добавлением объектов неправильного типа. Благодаря использованию TypeVar, коллекция может работать с любым типом данных, сохраняя при этом типобезопасность.

**TypeVar с ограничениями** — D и S, которые ограничивают типы элементов коллекции только теми, которые реализуют соответствующие Protocols. Это даёт возможность безопасно вызывать методы display() и score() внутри коллекции, зная что каждый элемент гарантированно имеет эти методы.

**Protocols Displayable и Scorable** — демонстрация структурной типизации. Объекты подходят под Protocol автоматически, если у них есть нужные методы, без явного наследования. Это делает код более гибким и позволяет использовать существующие классы без их модификации.

**Методы find, filter, map** — демонстрация работы с функциями высшего порядка в типизированной коллекции. Особенно показателен метод map, который меняет тип результата с помощью второго TypeVar R, позволяя преобразовывать коллекцию одного типа в коллекцию другого типа.
