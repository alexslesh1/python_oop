# Лабораторная работа №3
## Вариант №5: Транспортная система (наследование)

### Тема
Наследование и иерархия классов на примере автобусов

### Структура проекта
```
src/lab03/
├── base.py # Базовый класс Bus (из ЛР-1)
├── models.py # Дочерние классы (TouristBus, CityBus, SchoolBus)
├── demo.py # Демонстрационные сценарии
└── README.md # Описание
```

### Иерархия классов
```
Bus (базовый класс)
├── TouristBus (туристический)
├── CityBus (городской)
└── SchoolBus (школьный)
```


### Базовый класс Bus

| Атрибут | Описание |
|---------|----------|
| `_id` | Идентификатор автобуса |
| `_capacity` | Вместимость (1-150) |
| `_route_number` | Номер маршрута |
| `_driver_name` | Имя водителя |
| `_speed` | Текущая скорость (0-100) |
| `_status` | Состояние автобуса |
| `_last_maintenance` | Дата последнего ТО |

| Метод | Описание |
|-------|----------|
| `start_route()` | Выехать на маршрут |
| `park()` | Поставить на парковку |
| `send_to_maintenance()` | Отправить на ТО |
| `accelerate(inc)` | Увеличить скорость |
| `brake(dec)` | Уменьшить скорость |
| `calculate_fare()` | Базовая стоимость проезда (50 руб) |
| `get_bus_type()` | Возвращает тип автобуса |

### Дочерние классы

#### TouristBus (Туристический автобус)

| Новый атрибут | Описание |
|---------------|----------|
| `has_wifi` | Наличие Wi-Fi |
| `has_toilet` | Наличие туалета |
| `comfort_level` | Уровень комфорта (1-5) |

| Новый метод | Описание |
|-------------|----------|
| `serve_snacks()` | Подача закусков |

| Переопределенный метод | Значение |
|------------------------|----------|
| `calculate_fare()` | 50 + (комфорт × 50) + Wi-Fi(30) + туалет(40) |
| `get_bus_type()` | "Туристический автобус" |

#### CityBus (Городской автобус)

| Новый атрибут | Описание |
|---------------|----------|
| `has_validator` | Наличие валидатора билетов |
| `has_usb_ports` | Наличие USB-портов |
| `passengers_count` | Количество пассажиров в салоне |

| Новый метод | Описание |
|-------------|----------|
| `board_passenger()` | Посадка пассажира |
| `alight_passenger()` | Высадка пассажира |

| Переопределенный метод | Значение |
|------------------------|----------|
| `calculate_fare()` | 50 - 20 = 30 руб |
| `get_bus_type()` | "Городской автобус" |

#### SchoolBus (Школьный автобус)

| Новый атрибут | Описание |
|---------------|----------|
| `has_escort` | Наличие сопровождающего |
| `max_speed_limit` | Ограничение скорости (по умолчанию 70 км/ч) |
| `children_count` | Количество детей в салоне |

| Новый метод | Описание |
|-------------|----------|
| `board_child()` | Посадка ребенка |
| `play_educational_video()` | Включение обучающего видео |

| Переопределенный метод | Значение |
|------------------------|----------|
| `calculate_fare()` | 0 руб (бесплатно) |
| `get_bus_type()` | "Школьный автобус" |
| `accelerate()` | С учетом ограничения скорости |

### Полиморфные методы

| Метод | Bus | TouristBus | CityBus | SchoolBus |
|-------|-----|------------|---------|-----------|
| `calculate_fare()` | 50 руб | 250-370 руб | 30 руб | 0 руб |
| `get_bus_type()` | Обычный | Туристический | Городской | Школьный |

### Демонстрационные сценарии

| Сценарий | Что демонстрируется |
|----------|---------------------|
| **Сценарий 1** | Создание объектов всех типов (Bus, TouristBus, CityBus, SchoolBus) |
| **Сценарий 2** | Работа с методами дочерних классов (serve_snacks, board_passenger, board_child) |
| **Сценарий 3** | Полиморфизм (разное поведение calculate_fare) и проверка типов через isinstance |
| **Сценарий 4** | Фильтрация коллекции по типу (выборка только туристических/городских/школьных) |
| **Сценарий 5** | Уникальное поведение (ограничение скорости у школьного автобуса) |

### Интеграция с коллекцией из ЛР-2

Коллекция `BusFleet` успешно интегрирована и может хранить объекты всех типов, наследующихся от `Bus`. Это демонстрируется в сценариях 3 и 4.

### Пример использования
tourist_buses = filter_by_type(fleet, TouristBus)
city_buses = filter_by_type(fleet, CityBus)
school_buses = filter_by_type(fleet, SchoolBus)

### Вывод
В этой лабораторной работе я научился реализовывать иерархию классов с использованием наследования. Базовый класс Bus был расширен тремя дочерними классами: TouristBus, CityBus и SchoolBus. В каждом дочернем классе добавлены новые атрибуты и методы, а также переопределены полиморфные методы calculate_fare() и get_bus_type(). Созданная коллекция BusFleet из ЛР-2 успешно интегрирована и может хранить объекты всех типов. Реализована фильтрация по типу для выборки автобусов определенного класса. Также было продемонстрировано уникальное поведение школьного автобуса с ограничением скорости. Вся работа выполнена с использованием принципов ООП: наследование, инкапсуляция и полиморфизм.
