# Лабораторная работа №4: Интерфейсы и абстрактные классы (ABC)

## 1. Цель работы

В ходе лабораторной работы изучались:
- **Абстрактные базовые классы (ABC)** - классы, которые нельзя инстанциировать, они задают шаблон для дочерних классов
- **Интерфейсы** - контракты поведения, определяющие набор методов, которые класс обязан реализовать
- **Полиморфизм через интерфейсы** - вызов одинаковых методов на объектах разных типов
- **Множественное наследование** - реализация нескольких интерфейсов в одном классе

## 2. Описание интерфейсов

Создано три интерфейса (абстрактных класса) в файле `interfaces.py`:

| Интерфейс | Методы | Назначение |
|-----------|--------|-------------|
| **IPrintable** | `get_info() -> str` | Получение краткой информации об объекте для вывода |
| **IServiceable** | `needs_service() -> bool`<br>`service() -> str` | Проверка необходимости обслуживания и его выполнение |
| **IComparable** | `compare_to(other) -> int` | Сравнение объектов (-1, 0, 1) |

### Абстрактные методы
Все методы в интерфейсах помечены декоратором `@abstractmethod`, что обязывает дочерние классы реализовать их. При попытке создать экземпляр класса, не реализовавшего все абстрактные методы, возникнет ошибка.

## 3. Реализация в классах

На основе классов из ЛР-3 (`Bus`, `TouristBus`, `CityBus`, `SchoolBus`) созданы новые классы, реализующие интерфейсы:

| Класс | Реализуемые интерфейсы | Поведение методов |
|-------|------------------------|-------------------|
| **ServiceableBus** | `IPrintable`, `IServiceable`, `IComparable` | `get_info()`: стандартный формат<br>`needs_service()`: по `is_broken`<br>`compare_to()`: по вместимости |
| **ServiceableTouristBus** | `IPrintable`, `IServiceable`, `IComparable` | `get_info()`: с указанием Wi-Fi и комфорта<br>`needs_service()`: при поломке ИЛИ скорость >80<br>`compare_to()`: по уровню комфорта |
| **ServiceableCityBus** | `IPrintable`, `IComparable` | `get_info()`: с количеством пассажиров<br>`needs_service()`: не реализован<br>`compare_to()`: по загруженности (чем меньше, тем лучше) |
| **ServiceableSchoolBus** | `IPrintable`, `IServiceable` | `get_info()`: с количеством детей и сопровождением<br>`needs_service()`: по `is_broken`<br>`compare_to()`: не реализован |

### Отличия в поведении

1. **Метод `get_info()`** возвращает разную информацию в зависимости от типа автобуса
2. **Метод `needs_service()`** имеет разные условия:
   - Обычный автобус: только при поломке
   - Туристический: при поломке ИЛИ превышении скорости 80 км/ч
   - Школьный: только при поломке
   - Городской: данный метод отсутствует
3. **Метод `compare_to()`** использует разные критерии:
   - Обычный: по вместимости
   - Туристический: по уровню комфорта
   - Городской: по загруженности (меньше пассажиров = лучше)
